# Supabase M-11D-4 本人申請辞退UI 状態表示 実装結果

作業日: 2026-06-01

## 1. 実装範囲

M-11D-4として、`session-detail.html` の参加希望コメント欄に、本人申請状態に応じた申請取り下げUIの状態表示だけを追加した。

今回実装したもの:

- 本人申請状態メッセージ直下、投稿フォーム直上への取り下げUI器の追加。
- `pending` / `waitlisted` / `accepted` の場合の取り下げ導線表示。
- インライン確認UIの器表示。
- 確定ボタンのdisabled表示。
- `canceled` の「取り下げ済み」案内文への整理。
- `rejected` の申請不可案内維持。

今回実装していないもの:

- 申請取り下げRPC呼び出し。
- 実際の申請辞退。
- DB変更。
- Supabase SQL Editor実行。
- GM履歴RPC / UI。
- `close_session` 呼び出し。

## 2. UI配置

配置は `data-session-comment-post-control` 内の以下の順にした。

```text
本人申請状態メッセージ
申請取り下げUI器
投稿フォーム
```

コメントカード内の編集 / 削除操作とは分けた。コメント削除は発言を消す操作、申請取り下げはコメントを残して参加意思だけを外す操作として扱う。

## 3. 状態ごとの表示

`pending` / `waitlisted`:

```text
参加申請中です。
参加申請を取り下げる機能は次工程で実装予定です。
```

`accepted`:

```text
参加予定として承認済みです。
参加を取り下げる場合は、GMへの連絡も推奨されます。
取り下げ機能は次工程で実装予定です。
```

`canceled`:

```text
このセッションへの参加申請は取り下げ済みです。
再申請する場合は、参加希望コメントを投稿してください。
```

`rejected`:

```text
このセッションへの申請は現在行えません。
```

申請なしの場合は、既存どおり参加希望コメント投稿可能な表示を維持した。

## 4. 確認UIの器

`pending` / `waitlisted` / `accepted` では、`参加申請を取り下げる` ボタンでインライン確認UIを開ける。

確認UIには以下を表示する。

- 取り下げ確認文。
- コメントは履歴として残る旨。
- 次工程で実装予定である旨。
- `取り下げる（次工程で実装予定）` disabledボタン。
- `キャンセル` ボタン。

確定ボタンはdisabledで、申請取り下げRPCは呼び出さない。

## 5. 表示してよい情報 / 表示しない情報

表示してよい情報:

- 本人申請statusに応じた案内文。
- 参加申請取り下げUIの準備文言。
- 投稿フォーム、コメント本文、表示名、申請statusラベル。
- 申請中 / 承認済みカウント。

表示しない情報:

- email。
- `user_id` 全文。
- access token / refresh token / JWT。
- Project URL / key実値。
- secret類。
- `gmUserId`。
- `comment_id` / `application_id`。

## 6. 既存機能への影響

既存の投稿フォーム、コメント一覧、コメント新しい順、コメント編集、コメント削除、申請中 / 承認済みカウントは維持する方針で実装した。

`shouldShowCommentForm()` は `rejected` のみ投稿フォーム非表示のままとし、`canceled` は再申請用に投稿フォームを表示する。
