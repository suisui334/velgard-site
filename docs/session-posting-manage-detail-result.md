# M-14D-7 依頼書一覧のフォーム内格納＋編集モード器

## 実装概要

`session-post.html` のGM/admin向け `自分の依頼書` 一覧を、フォーム下部へ独立して長く並ぶカード列や右外側の独立パネルではなく、依頼書フォーム内の `公開状態` 欄の直下へ移した。

PC幅ではフォームグリッドの右列で `公開状態` の下に収まり、画面幅が狭い場合はフォーム内の自然な順序で下へ回り込む。
一覧は最大高さつきのスクロール領域にし、依頼書数が増えてもページ全体が過剰に縦長にならないようにした。

一覧に表示する情報:

- 募集状態バッジ
- 公開状態バッジ
- タイトル
- 開催日時から終了日時

概要や長文本文は一覧には表示しない。

## フォーム編集モード

一覧のタイトル行を選ぶと、同じ `session-post.html` のメインフォームへ依頼書内容を反映し、フォームを編集モード表示へ切り替える。

フォームに反映する情報:

- タイトル
- 開始日時
- 終了日時
- 申請締切
- 種別
- 募集人数min
- 募集人数max
- 公開状態
- 募集状態
- 概要

編集モード中は見出しを `編集中: 依頼書タイトル` にし、通常の新規作成モードと区別する。

## 誤新規作成防止

編集保存はまだ実装しない。

既存依頼書を選んだ状態で誤って重複新規作成しないよう、編集モード中は作成ボタンをdisabledにし、ボタン文言を `編集保存は次工程` に変える。
Enterなどでフォームsubmitが発生しても、編集モード中は `create_session_post(...)` を呼ばず、次工程予定メッセージを表示するだけにした。

## 新規作成へ戻る導線

編集モード中は `新規依頼書を書く` ボタンを表示する。
押すと選択状態を解除し、フォームを空に戻し、公開状態は `hidden`、募集状態は `draft`、種別は `one-shot` に戻す。
URLに `id` がある場合は取り除く。

新規作成モードでは従来どおりhidden/draftの新規作成が可能。

## 小さな詳細表示

フォーム内の管理ボックスに表示する詳細は最小限にした。

表示する情報:

- 公開状態
- 募集状態
- Discord同期状態
- 作成日時
- 更新日時

表示しない情報:

- `gm_user_id`
- email
- user_id全文
- gmUserId
- token / key / secret類
- Discord credential
- Webhook URL
- bot token
- service_role
